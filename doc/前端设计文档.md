  ## 1. 目标

  前端不再只是“看最终代码和播放器”，而是升级为一个可观测、可编辑、可局部重编译的 Video IDE。

  前端目标：

  - 展示两阶段流水线：博文 -> 口语稿 -> 视频编译
  - 展示完整中间产物：script / scenes / marks / layout / motion / dsl / code / validation
  - 支持 scene 级编辑与局部重编译
  - 支持失败定位、错误分类、修复提示
  - 与后端 compiler artifacts 强契约对齐
  - 保持 Remotion 预览稳定，不因单 scene 错误拖垮整条视频

  ## 2. 非目标

  本设计不覆盖：

  - UI 视觉品牌升级
  - 多人协作
  - 云端素材管理
  - 成片导出工作台
  - 商业化任务运营后台

  ## 3. 核心前端定位

  前端不是单纯播放器，而是三层系统：

  1. Pipeline IDE
  2. Artifact Inspector
  3. Remotion Runtime

  即：

  - 左侧管输入与工作流
  - 中间管预览与时间轴
  - 右侧管 artifacts / code / validation / patch

  ## 4. 核心用户流程

  ### 4.1 完整生成

  1. 用户输入博文
  2. 点击“生成口语稿”
  3. 前端显示 script workflow SSE 过程
  4. 用户确认或编辑口语稿
  5. 点击“生成视频”
  6. 前端显示 animation compiler 各阶段产物
  7. Remotion 预览生成结果
  8. 若某 scene 失败，只显示该 scene 错误，不影响其他 scene

  ### 4.2 单镜头重编译

  1. 用户选中某 scene
  2. 编辑文案、layout、motion 或 code
  3. 选择“从 layout/motion/dsl/code 重编译”
  4. 前端请求局部重编译接口
  5. 更新当前 scene 及后续关联产物

  ### 4.3 Debug

  1. 某 scene 渲染异常
  2. 前端显示错误阶段、错误码、scene_id、repairable
  3. 用户可执行 repair 或手动修改
  4. 修复后重新预览

  ## 9. API 对接设计

  ## 9.1 Script workflow

  接口：

  - POST /api/generate_script_sse

  前端处理：

  - 更新 pipelineStage = script_generating
  - 接收 setup/progress/updates/end/error
  - 将最终文案写入 oralScript

  ## 9.2 Animation workflow

  接口：

  - POST /api/generate_animation_sse

  输入应为：

  {
    "source_text": "<oral_script>",
    "thread_id": "optional"
  }

  前端语义上要明确：
  这里的 source_text 实际是口语稿，不再是原博文。

  ## 9.3 一键全流程

  建议新增：

  - POST /api/generate_video_pipeline_sse

  前端行为：

  - 先跑 script workflow
  - 再自动跑 animation workflow
  - UI 中清晰区分两个阶段

  ## 9.4 局部重编译

  接口：

  - POST /api/workflows/animation/regenerate_scene_sse
  - 后续建议新增更精细接口：
      - recompile_layout_sse
      - recompile_motion_sse
      - apply_patch_sse

  请求示例：

  {
    "thread_id": "xxx",
    "scene_id": "scene_2",
    "recompile_from": "layout"
  }

  ## 10. SSE 协议前端处理

  ### 10.1 事件类型

  前端需要支持以下事件：

  - setup
  - progress
  - stage_started
  - stage_completed
  - scene_started
  - scene_completed
  - validation_failed
  - repair_started
  - repair_completed
  - error
  - end

  ### 10.2 前端事件处理策略

  - setup：初始化 thread、重置状态
  - progress：更新顶部状态条
  - stage_started/completed：更新 workflow 节点 UI
  - scene_completed：写入 scene artifacts
  - validation_failed：写入 validation store
  - repair_*：更新 repair 状态
  - error：设置 pipeline error
  - end：按 success/partial_success/error 收尾

  ### 10.3 关键修正

  前端不能再把 end 默认当成功。
  必须读取：

  payload.status

  允许值：

  - success
  - partial_success
  - error

  - 当前 scene 文案
  - 当前 scene layout JSON
  - 当前 scene motion JSON

  ## 14.2 禁止直接编辑项

  v1 不允许前端直接改：

  - 编辑 motion：从 motion 重编译
  - 编辑 code：重新校验并立即预览

  ## 15. 错误处理设计

  ## 15.1 错误分类

  前端统一错误模型：

  type PipelineError = {
    type: 'compile' | 'agent' | 'render' | 'network';
    stage?: string;
    scene_id?: string;
    code?: string;
    message: string;
    repairable?: boolean;
  };

  ## 15.2 展示策略

  - 网络错误：顶部 toast + logs
  - 编译错误：Validation tab + SceneList 红点
  - 渲染错误：Preview fallback + code tab
  - repairable 错误：提供 repair 按钮

  ## 15.3 错误来源映射

  - 后端 validation_failed -> compile
  - 前端 react-live error -> render
  - SSE 断流 / fetch fail -> network

  ## 19. 验收标准

  ### 功能验收

  - 前端能分别跑 script workflow 和 animation workflow
  - 前端可展示至少 6 类中间产物
  - scene 级局部重编译可用
  - 单 scene 渲染失败不影响整体预览
  - validation 面板可用

  ### 质量验收

  - 同一 thread 重放能正确恢复 artifacts
  - SSE 中断不会污染 store
  - scene 切换时 artifacts 与 code 同步正确
  - timeline 与 marks 一致

  ### 工程验收

  - 类型定义完整
  - store action 清晰
  - SSE 处理逻辑统一复用
  - 组件职责边界清楚

  ## 20. 结论

  前端重构方向应该是：

  - 从“代码编辑器 + 播放器”升级为“可观测的 Pipeline IDE”
  - 从“只看最终代码”升级为“全链路 artifact inspection”
  - 从“全局刷新”升级为“scene 级局部重编译”
  - 从“黑盒报错”升级为“结构化错误与修复”
